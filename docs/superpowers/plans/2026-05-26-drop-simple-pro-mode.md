# Drop Simple/Pro Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the Simple/Pro mode toggle introduced in PR #3. Ship one unified UX: approachable Simple-style labels + plain English copy everywhere, but with dev affordances (Build .skill, Packages page, Bundle/Built-in skills, all Owner classes) always visible. No more mode branching.

**Architecture:** Three layers. (1) Rust: drop the `mode` and `mode_migration_announced` fields from `Settings`. (2) TypeScript foundation: flatten `copy.ts` to a single string map, delete `useMode` / `useCopy` / `ModeSwitch` / migration toast. (3) Per-component: strip `mode === "..."` branches, picking the Simple variant for labels/copy and the Pro variant for dev affordances (Bundle/Built-in visibility, hashes — wait, hashes go away; see Task 3.4 — Packages link, Build .skill button, full Owner badge set, build sha footer).

**Tech Stack:** Tauri 2, Rust (serde, ts-rs), React 18, TypeScript, TanStack Query.

**Worktree:** `/Users/devinwalker/Code/skill-sync/.worktrees/polish-gaps`
**Branch:** `claude/drop-simple-pro` (stacked on `claude/polish-gaps`, which is stacked on `claude/goofy-swirles-8b5480`)
